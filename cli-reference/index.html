    CLI Reference

    The pyzot provides a robust set of terminal commands under the zot binary.

    This reference is automatically generated from the application's Click source code using mkdocs-click.

    zot

    zot — CLI for your Zotero library.

    Usage:

    zot [OPTIONS] COMMAND [ARGS]...

    Options:

      --db TEXT                       Path to zotero.sqlite
  --library INTEGER               Library ID (default: 1)
  --format [table|json|csv]       Output format
  --no-color                      Disable colour output
  --allow-write                   Allow write operations (ad-hoc override; or
                                  set write.enabled=true in config).
  --connector-url TEXT            Zotero connector URL (default:
                                  http://127.0.0.1:23119).
  --require-zotero / --no-require-zotero
                                  Fail if Zotero is not reachable (default:
                                  require).
  --help                          Show this message and exit.

    add

    Add items to your Zotero library via the connector.

    When called with a single positional argument (no subcommand), automatically detects what kind of input it is (DOI, arXiv ID, PMID, ISBN, URL, citation string, or local file path) and dispatches to the right handler.

    Examples:

    zot add "10.1109/TPWRS.2023.1234567"        # auto-detect DOI
zot add "2401.12345"                        # auto-detect arXiv
zot add "Zhang, J. et al. (2025) Beyond..."  # auto-detect citation
zot add "/home/me/paper.pdf"                # auto-detect file
zot add batch papers.txt                    # batch mode

    Use zot add batch for scripted multi-item imports.

    Usage:

    zot add [OPTIONS] COMMAND [ARGS]...

    Options:

      --help  Show this message and exit.
    batch

    Process a file of inputs, one per line.

    FILE may be - to read from stdin. Each line is passed through the same auto-detect dispatcher as zot add "<anything>".

    Lines that start with # or are blank are silently skipped.

    After processing all lines a summary table is printed and, if any lines failed, the command exits with code 1.

    Note: --jobs N (N > 1) is reserved for parallel resolver lookups in a future release. Currently it is accepted but has no effect.

    Example:

    zot add batch papers.txt --collection "Smart Grid"
cat dois.txt | zot add batch - --tag imported

    Usage:

    zot add batch [OPTIONS] FILE

    Options:

      -c, --collection NAME           Collection name applied to all items.
  -t, --tag TEXT                  Tag to apply to all items (repeatable).
  --dry-run                       Resolve and print what would be submitted
                                  without making connector calls.
  --on-duplicate [report|skip|force-add]
                                  Behaviour when a duplicate is found.
                                  [default: report]
  -v, --verbose                   Print verbose HTTP request/response info.
  --non-interactive               Never prompt; fail/skip ambiguous citations.
  --jobs INTEGER                  (Stub — reserved for parallel resolver
                                  lookups in v0.3.0; currently no-op.)
                                  [default: 1]
  --help                          Show this message and exit.
    cite

    Add one or more items from free-text citation strings.

    Usage:

    zot add cite [OPTIONS] [CITATION_TEXT]

    Options:

      -f, --file PATH
  --threshold INTEGER             [default: 50]
  --gap FLOAT                     [default: 1.4]
  -c, --collection NAME           Collection name to add the item to.
  -t, --tag TEXT                  Tag to apply (repeatable).
  --dry-run                       Print the JSON that would be POSTed without
                                  making any request to the connector.
  --on-duplicate [report|skip|force-add]
                                  Behaviour when a duplicate is found.
                                  [default: report]
  -v, --verbose                   Print verbose HTTP request/response info.
  --non-interactive               Never prompt for citation disambiguation.
  --help                          Show this message and exit.

    Usage:

    Options:

      --help  Show this message and exit.

    attachments

    Browse and open Zotero attachments.

    Usage:

    zot attachments [OPTIONS] COMMAND [ARGS]...

    Options:

      --help  Show this message and exit.
    add

    Attach an existing local file to an existing Zotero item.

    Inserts the attachment directly into zotero.sqlite and copies the file into <zotero-data-dir>/storage/<new-key>/.

    Example:

    zot attachments add ABCD1234 ~/Downloads/paper.pdf

    Usage:

    zot attachments add [OPTIONS] PARENT_KEY FILE_PATH

    Options:

      --title TEXT       Display title (default: filename stem).
  --source-url TEXT  Source URL for provenance.
  --help             Show this message and exit.
    fetch

    Find and attach a PDF for an existing item using HTTP resolvers.

    Tries DOI, item URL, and any custom resolvers in turn. The first successful PDF is attached as a child of the item.

    Example:

    zot attachments fetch 5UFZMSLU

    Usage:

    zot attachments fetch [OPTIONS] PARENT_KEY

    Options:

      --methods TEXT  Comma-separated subset of resolvers to enable.  [default:
                  doi,url,custom]
  --help          Show this message and exit.
    fetch-all

    Find PDFs for every library item that has a DOI/URL but no PDF.

    Long-running. Honour --limit while testing. Items already with a PDF attachment are skipped. Rate-limiting is enforced per-domain by the pipeline internally.

    Example:

    zot attachments fetch-all --limit 20

    Usage:

    zot attachments fetch-all [OPTIONS]

    Options:

      --methods TEXT   [default: doi,url,custom]
  --limit INTEGER  Cap total items processed.
  --help           Show this message and exit.
    fetch-collection

    Find PDFs for every item in a collection that doesn't already have one.

    Example:

    zot attachments fetch-collection "[Paper] LV_UG_Cable_Models_DSSE"

    Usage:

    zot attachments fetch-collection [OPTIONS] COLLECTION_NAME

    Options:

      --methods TEXT      [default: doi,url,custom]
  --include-with-pdf  Also process items that already have a PDF (re-fetch).
  --help              Show this message and exit.

    search

    Search items across fields, authors, tags, DOIs, or years.

    Examples: zot search "bayesian" --field title zot search --author "Numair" zot search "power flow" --type journalArticle zot search --doi 10.1038/example zot search --tag "machine-learning" zot search --year 2020-2024

    Usage:

    zot search [OPTIONS] [QUERY]

    Options:

      --field TEXT   Restrict search to field(s) e.g. title, abstract
  --author TEXT  Search by author name (first or last, partial match)
  --type TEXT    Restrict to item type
  --doi TEXT     Search by DOI
  --tag TEXT     Search by tag
  --year TEXT    Year range e.g. 2020-2024 or 2022
  --fulltext     Use full-text index instead of field search
  --help         Show this message and exit.
